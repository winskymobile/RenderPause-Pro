# RenderPause Pro 使用说明

**版本：** v1.0.1  
**系统：** macOS 26+ · Apple Silicon  
**一句话：** 解决 M 系列芯片 macOS 闲时 GPU 占用过高；正常使用无感，只在后台真闲置时动手。

---

## 前言

闲时 GPU 偏高，很多时候并不是什么玄学：  
**把暂时不用的窗口最小化或隐藏**，WindowServer 往往就会停掉对它的持续合成，GPU 会跟着下来。

问题在于习惯。

多数人切应用时只是点到别的窗口继续工作，**很少会主动最小化**那些已经用完、被挡住的应用。窗口一多，后台图层仍在被合成——人已经在干别的事，机器却还在为看不见的界面费电、发热。

所以才有 RenderPause Pro：  
不是发明一种新的「底层禁渲染」魔法，而是把「该最小化 / 隐藏时自动做掉」这件事，做成可配置、可恢复、尽量不打扰的后台服务。你继续按自己的节奏用 Mac；它只在名单应用真正闲置时补上这一步。

---

## 1. 为什么做这个软件

### 1.1 核心目标：降闲时 GPU，不改日常习惯

RenderPause Pro **不是**加速前台、也不是「一键清理」。  
你日常打开应用、切换窗口、写作绘图，**和平时一模一样**。

它只解决一件事：

> 你已经不用某个窗口了，它却还在后台被系统持续合成，导致 **闲时 GPU 偏高**——而你通常也不会手动去最小化它。

在 Apple Silicon 上很常见：

- 已经切去写代码、看网页，Pencil / 腾讯文档 / Discord 等只是挂在后面；
- 活动监视器里 **WindowServer** 或整体 GPU 仍然不低；
- 发热、风扇、续航变差——**人在忙别的，GPU 还在给看不见的窗口干活**。

根因往往不是「又在跑大模型」，而是后台图层仍在合成。统一内存下，这类开销更容易传导到整机。

### 1.2 系统边界

在 macOS 上**不能**像部分 Windows 工具那样：

- 注入第三方进程、挂钩 Metal Command Buffer；
- 用私有 API 强行给合成器降帧。

这些属于 SIP / 驱动特权区，第三方做不到，也不应做。

### 1.3 有效且合规的做法

实践中有效、且走公开能力的路径是：

1. **隐藏**应用（类似 `Cmd+H`）→ WindowServer 停止对其图层的持续合成；  
2. 或 **最小化**窗口（需辅助功能）→ 同类效果。  

实测：非活跃时隐藏 / 最小化，闲时 GPU 可明显回落——这和你**手动**点黄灯或 `Cmd+H` 是同一条路径。  
本工具只是替你自动完成这一步：**只对你主动加入的名单**、在真正「闲置且完全看不见」时动手；切回立刻恢复。  
**正在用的前台应用不会被动。**

**产品目标：** 在不影响正常使用的前提下，补上「忘了最小化」的空档，降低闲时无效合成带来的 GPU 占用。

## 2. 使用前先做这两步（很重要）

这两步决定你会不会觉得「有用」——也是本工具和全家桶清理软件最大的差别。

### 2.1 确认问题

打开 **「活动监视器」→ GPU / WindowServer**，在「窗口很多但你在干别的事」时观察是否偏高。

- **偏高**：闲时合成很可能就是元凶，继续下一步。  
- **不高**：问题可能在别处（别的进程、充电/性能模式等），不必硬用本工具。

### 2.2 只加真凶

偏好设置 → **应用名单** → **添加**，只加入你怀疑后台仍费 GPU 的应用（例如你实测挂着就飙 WindowServer 的那几个）。

- **不必全加**。没进名单的应用，本工具**完全不碰**。  
- 加得越精，误伤越少，也越容易对照活动监视器验证效果。

## 4. 它做什么 / 不做什么

### 4.1 做

| 能力 | 说明 |
|------|------|
| Opt-in 名单 | 默认空；只有你添加的应用才会被管，其余完全不动 |
| 自动隐藏 | 离开前台 + 完全遮挡 + 达到触发时间 → 隐藏 |
| 立即恢复 | 切回前台 / 退出本工具时恢复显示 |
| 防误伤 | 部分可见不藏、分屏搭档保护、忽略输入法前台 |
| 监控总开关 | 一键暂停全部优化（需要时随时关掉） |
| 最近活动 | 记录何时对谁做了优化 / 恢复 |

### 4.2 不做

- 不注入、不改第三方渲染代码  
- 不 `SIGSTOP` 挂起进程（避免中断 AI / 音频 / 未保存状态）  
- 不扫描「全盘加速」、不展示夸张省电百分比  
- 不改变你正在使用的前台体验  
- 当前界面**不提供**最小化策略开关（代码保留，默认强制隐藏）

## 5. 安装与启动

### 5.1 构建

需要 Xcode 26+、Apple Silicon；可选 XcodeGen。

```bash
cd "/path/to/RenderPause Pro"
xcodegen generate
xcodebuild -scheme RenderPausePro \
  -destination 'platform=macOS' \
  -configuration Debug \
  build
```

### 5.2 安装运行

```bash
APP=$(ls -d ~/Library/Developer/Xcode/DerivedData/RenderPausePro-*/Build/Products/Debug/RenderPausePro.app | head -1)
pkill -x RenderPausePro 2>/dev/null || true
cp -R "$APP" /Applications/RenderPausePro.app
open /Applications/RenderPausePro.app
```

- 菜单栏出现图标，**没有** Dock 图标（Agent 形态）。  
- 首次可能出现欢迎引导。  

### 5.3 退出

菜单栏 → **退出**。退出前会恢复仍处于「已隐藏」状态的名单应用。  
偏好窗 `⌘W` 只关窗；`⌘Q` 在偏好窗前台时先关窗，无窗时才退出进程。

## 6. 推荐使用流程（针对闲时 GPU）

先完成 **[§2 使用前先做这两步](#2-使用前先做这两步很重要)**（确认问题 + 只加真凶），再按下面操作。

1. **打开监控**  
   「启用监控」打开；触发时间先用 **30 秒**（可 5–600）。

2. **照常使用**  
   继续正常工作。只有名单应用被完全挡住并超过阈值时，才会自动隐藏。  
   再看 GPU / WindowServer：预期闲时回落。

3. **需要时再切回**  
   `Cmd+Tab` 或点击 Dock 图标切回，窗口恢复显示，无需手动「找隐藏应用」。

4. **临时关掉**  
   演示、录屏、不希望任何自动隐藏时：菜单栏 **暂停监控**。暂停后行为与未安装时一致。

| **已关闭** | 在名单但开关关，不参与 |

## 8. 触发规则（引擎逻辑）

### 8.1 隐藏条件（全部满足）

1. 监控总开关 = 开  
2. Bundle ID 在名单且规则启用  
3. 该应用不是当前「常规前台」  
4. 窗口几何上**完全遮挡**（露出一点 → 不隐藏）  
5. 处于上述状态的时间 ≥ 触发秒数  
6. 不是分屏（Split View）中被保护的搭档应用  

不满足任一条时：**不动作**，与未安装本工具时相同。

### 8.2 恢复条件

- 用户重新激活该应用  
- 退出 RenderPause Pro  

### 8.3 为何强调「完全遮挡」

闲时 GPU 问题来自「看不见却仍在合成」。  
若窗口还露一角，你可能仍在扫一眼内容——此时隐藏会打扰使用，也省不了多少。  
**完全遮挡**才更接近「真闲置」，和「正常使用无感、只降闲时 GPU」的目标一致。

## 9. 权限与隐私

| 能力 | 是否需要额外授权 |
|------|------------------|
| 隐藏（主功能） | 否 |
| 最小化（未对用户开放） | 是 · 辅助功能 |
| 登录时启动 | 系统登录项 |
| 前台 / 遮挡检测 | 公开 API，不需「输入监控」 |

- 不注入第三方 App  
- 不读取 App 内文档或账号数据  
- 设置与名单存在本机用户偏好（`com.renderpause.pro`）  

功能开关：`FeatureFlags.allowMinimizeMode`（当前 `false`）控制是否露出最小化相关 UI。

## 10. 故障排查

| 现象 | 排查 |
|------|------|
| GPU 仍然高 | 是否加对了应用？是否完全遮挡？监控是否开启？是否到时间？用活动监视器看是不是别的进程 |
| 一直不隐藏 | 窗口露出一点、分屏搭档、规则关闭、监控暂停 |
| 正在用却被藏 | 确认是否真的是前台；升级后应有输入法/遮挡保护；可先暂停监控 |
| 隐藏了但 GPU 降得少 | 还有其他后台大户；或系统其它进程 |
| 菜单/图标异常 | 重启 App；必要时注销登录 |
| 登录启动失败 | 系统设置 → 通用 → 登录项 中检查 |

## 11. 和「清理类软件」的差别

| | 常见加速/清理软件 | RenderPause Pro |
|--|-------------------|-----------------|
| 卖点 | 一键加速、大数字 | 降**闲时合成**，日常使用无感 |
| 范围 | 倾向全家桶 | 仅 opt-in 名单 |
| 手段 | 各异，偶有激进 | 系统级隐藏，可恢复 |
| 界面 | 仪表盘、恐吓文案 | 系统设置式克制 UI |

成功标准：**正常使用不被打扰、恢复稳、闲时 GPU 下来**，而不是首页刷一串「已优化帧数」。

## 13. 已知限制

1. 目标平台为 Apple Silicon + macOS 26+。  
2. 遮挡为几何近似，极端多桌面 / Stage Manager 场景可再增强。  
3. 部分 Electron 对 hide 返回值异常，以系统 `isHidden` 为准。  
4. 未走 Mac App Store 沙盒与公证。  
5. 不能消灭所有闲时 GPU（驱动、其它 App、系统动画仍在）。本工具只收「名单窗口无效合成」这一刀。
