# RenderPause Pro 使用说明

**版本：** v1.0.0  
**系统：** macOS 26+ · Apple Silicon  
**一句话：** 专门对付「闲着不用时 GPU / WindowServer 仍然很高」的菜单栏工具。

---

## 1. 为什么做这个软件

### 1.1 核心痛点：闲时 GPU 过高

在 Apple Silicon Mac 上，下面这种情况很常见：

- 你已经切去写代码、看网页，Pencil / 腾讯文档 / Discord / 部分 Electron 应用只是**挂在后面**；
- 活动监视器里 **WindowServer** 或整体 GPU 仍然不低；
- 机器发热、风扇响、续航掉得快——**人在闲，GPU 不闲**。

根因往往不是「又在跑大模型」，而是：

> 那些窗口虽然你不看了，系统仍在为它们做**图层合成**。  
> 统一内存架构下，App 与 WindowServer 的 GPU 负担更容易互相传导，静止窗口越多，闲时成本越明显。

### 1.2 系统边界：不能硬刚渲染循环

在 macOS 上**不能**像部分 Windows 工具那样：

- 注入第三方进程、挂钩 Metal Command Buffer；
- 用私有 API 强行给合成器降帧。

这些属于 SIP / 驱动特权区，第三方做不到，也不应做。

### 1.3 有效且合规的做法

实践中有效、且走公开能力的路径是：

1. **隐藏**应用（类似 `Cmd+H`）→ WindowServer 停止对其图层的持续合成；  
2. 或 **最小化**窗口（需辅助功能）→ 同类效果。  

实测：非活跃时隐藏/最小化，GPU 可明显回落。  
RenderPause Pro 把这件事自动化：只对名单应用、在真正「闲置且看不见」时动手，切回立刻恢复。

**产品目标：** 降低**闲时**无效合成带来的 GPU 占用，而不是加速你正在用的前台应用。

## 2. 它做什么 / 不做什么

### 2.1 做

| 能力 | 说明 |
|------|------|
| Opt-in 名单 | 默认空；只有你添加的应用才会被管 |
| 自动隐藏 | 离开前台 + 完全遮挡 + 达到触发时间 → 隐藏 |
| 立即恢复 | 切回前台 / 退出本工具时恢复显示 |
| 防误伤 | 部分可见不藏、分屏搭档保护、忽略输入法前台 |
| 监控总开关 | 一键暂停全部优化 |
| 最近活动 | 记录何时对谁做了优化/恢复 |

### 2.2 不做

- 不注入、不改第三方渲染代码  
- 不 `SIGSTOP` 挂起进程（避免中断 AI/音频/未保存状态）  
- 不扫描「全盘加速」、不展示夸张省电百分比  
- 当前界面**不提供**最小化策略开关（代码保留，默认强制隐藏）

## 3. 安装与启动

### 3.1 构建

需要 Xcode 26+、Apple Silicon；可选 XcodeGen。

```bash
cd "/path/to/RenderPause Pro"
xcodegen generate
xcodebuild -scheme RenderPausePro \
  -destination 'platform=macOS' \
  -configuration Debug \
  build
```

### 3.2 安装运行

```bash
APP=$(ls -d ~/Library/Developer/Xcode/DerivedData/RenderPausePro-*/Build/Products/Debug/RenderPausePro.app | head -1)
pkill -x RenderPausePro 2>/dev/null || true
cp -R "$APP" /Applications/RenderPausePro.app
open /Applications/RenderPausePro.app
```

- 菜单栏出现图标，**没有** Dock 图标（Agent 形态）。  
- 首次可能出现欢迎引导。  

### 3.3 退出

菜单栏 → **退出**。退出前会恢复仍处于「已隐藏」状态的名单应用。  
偏好窗 `⌘W` 只关窗；`⌘Q` 在偏好窗前台时先关窗，无窗时才退出进程。

## 4. 推荐使用流程（针对闲时 GPU）

1. **确认问题**  
   打开「活动监视器」→ GPU / WindowServer，在「窗口很多但人在摸鱼」时观察是否偏高。

2. **只加真凶**  
   偏好设置 → 应用名单 → **添加**，加入你怀疑后台仍费 GPU 的应用（如 Pencil、腾讯文档）。  
   不要一股脑全加，避免无谓打扰。

3. **打开监控**  
   「启用监控」打开；触发时间先用 **30 秒**（可 5–600）。

4. **复现闲时场景**  
   把名单应用切到后面，用别的全屏/大窗**完全挡住**它们，等待触发时间。  
   再看 GPU / WindowServer：预期回落。

5. **继续工作**  
   需要时再切回，窗口会恢复，无需手动找隐藏应用。

6. **临时关掉**  
   演示、录屏、不希望任何自动隐藏时：菜单栏 **暂停监控**。

## 5. 界面说明

### 5.1 菜单栏

```
监控中
后台 30 秒 · 今日 12 次
────────────────
暂停监控
────────────────
应用名单
✓  腾讯文档     已隐藏
✓  Pencil       监控中
   Figma        已关闭
────────────────
打开 RenderPause Pro   ⌘,
退出                   ⌘Q
```

| 状态字 | 含义 |
|--------|------|
| **已隐藏** | 已因闲时策略隐藏（对应「正在帮你省合成」） |
| **监控中** | 已启用，尚未触发或已恢复 |
| **已关闭** | 在名单但开关关，不参与 |

点击名单行 = 切换该应用是否参与优化。勾选为纯 ✓、无外圈。

### 5.2 偏好设置

- 默认窗口 **740×500**（最小可缩放尺寸，可放大）。  
- 标题：`RenderPause Pro v1.0.0`。  

**左 · 通用**

- 启用监控  
- 登录时启动  
- 触发时间（秒）：完全遮挡后等多久再隐藏  

**左 · 最近活动**

- 优化 / 恢复 / 错误日志；今日次数  

**右 · 应用名单**

- 勾选 · 图标 · 名称 · **状态** · 垃圾桶移除  
- **添加**：从运行中应用多选加入

## 6. 触发规则（引擎逻辑）

### 6.1 隐藏条件（全部满足）

1. 监控总开关 = 开  
2. Bundle ID 在名单且规则启用  
3. 该应用不是当前「常规前台」  
4. 窗口几何上**完全遮挡**（露出一点 → 不隐藏）  
5. 处于上述状态的时间 ≥ 触发秒数  
6. 不是分屏（Split View）中被保护的搭档应用  

### 6.2 恢复条件

- 用户重新激活该应用  
- 退出 RenderPause Pro  

### 6.3 为何强调「完全遮挡」

闲时 GPU 问题来自「看不见却仍在合成」。  
若窗口还露一角，用户可能仍在扫一眼内容——此时隐藏会误伤，也省不了多少。  
**完全遮挡**才更接近「真闲置」，和本工具要打的 GPU 场景一致。

## 7. 权限与隐私

| 能力 | 是否需要额外授权 |
|------|------------------|
| 隐藏（主功能） | 否 |
| 最小化（未对用户开放） | 是 · 辅助功能 |
| 登录时启动 | 系统登录项 |
| 前台 / 遮挡检测 | 公开 API，不需「输入监控」 |

- 不注入第三方 App  
- 不读取 App 内文档或账号数据  
- 设置与名单存在本机用户偏好（`com.renderpause.pro`）  

功能开关：`FeatureFlags.allowMinimizeMode`（当前 `false`）控制是否露出最小化相关 UI。

## 8. 故障排查

| 现象 | 排查 |
|------|------|
| GPU 仍然高 | 是否加对了应用？是否完全遮挡？监控是否开启？是否到时间？用活动监视器看是不是别的进程 |
| 一直不隐藏 | 窗口露出一点、分屏搭档、规则关闭、监控暂停 |
| 正在用却被藏 | 确认是否真的是前台；升级后应有输入法/遮挡保护；可先暂停监控 |
| 隐藏了但 GPU 降得少 | 还有其他后台大户；或系统其它进程 |
| 菜单/图标异常 | 重启 App；必要时注销登录 |
| 登录启动失败 | 系统设置 → 通用 → 登录项 中检查 |

## 9. 和「清理类软件」的差别

| | 常见加速/清理软件 | RenderPause Pro |
|--|-------------------|-----------------|
| 卖点 | 一键加速、大数字 | 针对**闲时合成** |
| 范围 | 倾向全家桶 | 仅 opt-in 名单 |
| 手段 | 各异，偶有激进 | 系统级隐藏，可恢复 |
| 界面 | 仪表盘、恐吓文案 | 系统设置式克制 UI |

成功标准：**误伤少、恢复稳、闲时 GPU 下来**，而不是首页刷一串「已优化帧数」。

## 11. 已知限制

1. 目标平台为 Apple Silicon + macOS 26+。  
2. 遮挡为几何近似，极端多桌面 / Stage Manager 场景可再增强。  
3. 部分 Electron 对 hide 返回值异常，以系统 `isHidden` 为准。  
4. 未走 Mac App Store 沙盒与公证。  
5. 不能消灭所有闲时 GPU（驱动、其它 App、系统动画仍在）。本工具只收「名单窗口无效合成」这一刀。
